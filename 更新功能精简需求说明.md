# Proxy Control Mobile

## 更新功能精简需求说明（Update Feature Minimal Spec）

### 1. 目标

为当前项目增加一个 **GitHub 更新检查与手动更新功能**，允许用户在网页控制台中：

* 查看当前运行版本
* 检查 GitHub 是否有新版本
* 手动执行 `git pull` 更新
* 可选更新后重启服务

该功能必须 **安全、简单、可回滚、不影响代理链运行**。

---

# 2. 功能范围

本功能包含三个能力：

1. **检查更新**
2. **执行更新**
3. **更新后重启服务**

不实现：

* 自动更新
* 自动定时更新
* Git merge/rebase
* 多分支管理

仅支持：

```
origin/main
```

---

# 3. 新增 API

新增以下 HTTP API。

---

## 3.1 检查更新

```
GET /api/update/status
```

返回：

```json
{
  "ok": true,
  "data": {
    "repo": "https://github.com/xxx/proxy-control",
    "branch": "main",
    "local_commit": "a1b2c3d",
    "remote_commit": "e4f5g6h",
    "update_available": true,
    "last_check": "2026-03-08T12:00:00"
  }
}
```

逻辑：

执行：

```
git rev-parse HEAD
git rev-parse --abbrev-ref HEAD
git fetch origin
git rev-parse origin/main
```

如果：

```
local_commit != remote_commit
```

则：

```
update_available = true
```

说明：
需要先 `git fetch` 才能获取远端最新 commit。 ([GeeksforGeeks][1])

---

# 3.2 手动检查更新

```
POST /api/update/check
```

行为：

```
git fetch origin
```

然后返回与 `/api/update/status` 相同结构。

---

# 3.3 执行更新

```
POST /api/update/pull
```

执行：

```
git pull --ff-only origin main
```

说明：

使用 `--ff-only` 避免 merge 冲突。

返回：

```json
{
  "ok": true,
  "message": "updated",
  "data": {
    "old_commit": "a1b2c3d",
    "new_commit": "e4f5g6h"
  }
}
```

如果没有更新：

```
message: "already up to date"
```

---

# 3.4 更新并重启

```
POST /api/update/pull_restart
```

行为：

```
git pull --ff-only origin main
```

然后执行：

```
./stop_full.sh
./start_full.sh
```

返回：

```json
{
  "ok": true,
  "message": "updated and restarted"
}
```

# 6. 安全限制

必须实现以下限制：

### 1 不允许 dirty working tree

执行：

```
git status --porcelain
```

如果非空：

返回：

```
error: working tree not clean
```

防止覆盖本地修改。

---

### 2 仅允许 main 分支

检查：

```
git branch --show-current
```

如果不是：

```
main
```

拒绝更新。

---

### 3 更新超时

所有 git 命令必须：

```
timeout 20s
```

避免 SSH/Git 卡死。
